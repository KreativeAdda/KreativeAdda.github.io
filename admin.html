How public uploads work

GitHub Pages is a static website. Photos and videos become visible to everyone only after they are uploaded to the GitHub repository. Browser-only uploads stay on one laptop, so they were removed from the live workflow.

Add photos

Upload image files to assets/uploads/, then add an item in shared.js like this:

{
  title: "My Dish",
  category: "dishes",
  type: "image",
  src: "assets/uploads/my-dish.jpg",
  caption: "A small note about this photo."
}

Add YouTube videos

Add the YouTube link in shared.js. Visitors can watch it inside the website:

{
  title: "My Song",
  category: "sargam",
  type: "youtube",
  youtube: "https://www.youtube.com/watch?v=VIDEO_ID",
  caption: "A short description."
}

Update word, thought, contact and links

Edit the daily and profile sections inside shared.js. This is the reliable GitHub Pages way unless a separate cloud backend is built.
